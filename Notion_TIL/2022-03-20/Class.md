### 클래스란

### 클래스의 특징

클래스(Class)는 복사본을 만들기 위한 원형(Prototype)이다.

- 각자 따로 보관하고 싶어하는 데이터 혹은 속성들을 정의
- 데이터에 관계없이 제공되는 같은 기능이나 로직

변수입니다(객체마다 보관하는 변수라는 의미에서 인스턴스 변수)

**클래스의 형태** 

`클래스 이름 {
데이터를 담을 변수1
데이터를 담을 변수2
...
데이터를 담을 변수n`

`어떤 로직을 수행하는 메소드1
어떤 로직을 수행하는 메소드2
...
어떤 로직을 수행하는 메소드n`

`}`

### 객체자료형 : 객체의 레퍼런스(리모컨)을 담는 변수이다

JAVA에서 변수는 두 가지 스타일로 나누어서 구분한다.

- 기본 자료형 : 변수에 데이터만 담긴다.
    - 기본형 자료는 상자의 내용물로 데이터가 들어갑니다.
    - 기본값을 가지는 값들
        - byte, short, int, long: 숫자이므로 0
        - double, float: 소수이지만 데이터가 없으므로 0.0
        - char: 아스키코드에 따라서 ' ' (공백 문자)
        - boolean: false
- 객체 자료형 : 변수에 리모콘이 담긴다.
    - 객체자료형은 리모컨을 저장할 수 있는 공간만 만든다. 이때 상자에 텅빈 리보컨이 생성되는데, 이것이 바로 null값이다.
        
        null이라는 존재는 상당히 애매모호하다. (객체 자료형도 아니고 기본 자료형도 아닌) 
        
        하지만 null에 대해서는 다음과 같은 사항을 알면 프로그램을 작성할 때 조금 도움이 됩니다.
        
        - 제어문에서 변수에 리모컨이 실제 객체를 가리키는 데 사용한다.
            - if(a == null)이라는 연산을 한다는 것은 a가 객체 자료형이라는 것이고, a 안의 리모컨이
            가리키는 객체가 있는지 없는지를 따질 때 사용합니다.
        - 객체 자료형 변수의 상자에 내용물이 없을 때 사용한다.
        
        ### NullPointerException이라는 에러
        
        ![Untitled](2%20Class%20acad6582dfe546bd8d1ec8a2a8d2bc33/Untitled%202.png)
        
        ### NullPointerException을 해결하는 가장 쉬운 비법
        
        - 가장 쉬운 방법은 에러 발생하는 위치의 코드에서 dot(.)을 사용하는 부분을 찾아내는 것이다.
        - NullPointerException은 상자 안에 담긴 리모컨이 종이 리모컨(null)인데 이것을 누르면서 실제로 움직이기를 원할 때 발생하는 겁니다.
        
        ### 객체의 데이터(인스턴스 변수)
        
        - 인스턴스 변수는 클래스의 복사본인 각각의 객체마다 가지는 데이터를 의미합니다. 같은 MP3가 여러 대있다면 MP3마다 가지는 노래 데이터가 바로 인스턴스 변수가 됩니다.
        - 클래스의 { } 안쪽에 선언한 변수는 '객체마다 가지는 데이터
        - 클래스에서 생산된 객체마다 독립적으로 그 데이터를 가지고 있다는 뜻
        
        <aside>
        💡 인스턴스 변수는 필요하다면 클래스에서 나온 모든 객체마다 다른 값을 가질 수도 있습니다(예를 들어 여
        러분의 휴대전화 번호를 생각하시면 됩니다). 따라서 인스턴스 변수는 필요하지 않다면 굳이 변수의 값을
        지정하지 않아도 됩니다.
        
        </aside>
        
        ### 인스턴스 변수는 언제 사용되는가?
        
        1. 메소드의 실행결과(실행된 데이터)를  누적되어서 보관하는 역할로 사용된다. 
        2. 객체의 레퍼런스(리모컨)으로 사용할 수 있다. 객체의 메소드를 실행하는 것과 객체의 데이터(속성)에 접근하는 작업입니다.
        
        ### .(dot)
        
        JAVA 프로그래밍에서 어디에선가 접근하거나 메소드를 실행할 때 사용한다.
        
        - 객체가 각진 메소드
        - 객체가 가직 데이터
        
        ### 캡슐화(Encapsulation) or 정보 은닉: 객체의 데이터는 함부로 공개하지 않는다.
        
        객체의 데이터를 마음대로 접근할 수 있다면 메소드를 통해서 만들어진 데이터는 의미가 없게 되는 문제
        가 생깁니다. 예를 들어 입금하지 않아도 저금통 안의 데이터를 변경할 수 있다면 total 데이터의 값을 사
        용자 마음대로 조작할 수 있게 되는 문제가 생깁니다.
        
        객체의 데이터를 보호한다는 의미는 이 금액 데이터에 직접 접근하면 안된다는 것이다. 
        
        객체에서 데이터는 어떤 메소드의 실행 결과를 누적해서 보존하는 경우가 많기 때문에 함부로 접근할 수 있게 해주면 안된다. 
        
        클래스를 작성할 때 인스턴스 변수를 private으로 작성하는 것의 의미는 
        
        클래스의 { } 를 벗어나서는 접근이 불가능하다라는 뜻

        ### private으로 처리된 객체의 데이터가 필요한데 어떻게 접근해야하지?
        
        ⇒ Getter 메소드를 사용해라
        
         가장 중요한 것은 Java에서의 변수 할당은 '복사(Copy)'라는 개념이다. 데이터에 직접 접근하지 않고 데이터를 복사하기 때문에 원본은 안전해진다. 
        
        ![Untitled](2%20Class%20acad6582dfe546bd8d1ec8a2a8d2bc33/Untitled%203.png)
        
        getter 데이터를 얻어오는 기능
        
        ### 메소드를 통해 데이터를 변경하려면 Setter
        
        메소드를 통해서만 해당 데이터에 접근이 가능하다.
        
        접근이 가능하다는 말은 데이터도 바꿀 수 있기 때문에 setter는 반드시 신중하게 생각하고 필요한 경우에만  작성해야한다.
